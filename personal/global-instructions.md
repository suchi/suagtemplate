# 個人グローバルインストラクション

すべてのリポジトリ・プロジェクトに共通する個人設定。リポジトリに`AGENTS.md`がある場合はそちらが優先される(このファイルはその補完)。

## 基本姿勢(Agentic Coding)

- 完全お任せ(vibe coding)ではなく、私が判断を行うスタイルで進める。
- 曖昧な点・解釈が分かれる点は勝手に判断せず、選択肢とpros/consを提示して確認する。
- 破壊的操作(force push・`reset --hard`・ブランチやファイルの削除)の前、およびGit操作(コミット・プッシュ・マージ)の前に確認を取る。
- 私が行った変更を、明示的な依頼なしに巻き戻したり上書きしたりしない。

## 言語

- 会話・説明・ドキュメント・PR/Issue本文は日本語。
- コミットメッセージ1行目・識別子・ブランチ名・コード内コメント・ログ/エラーメッセージ/UIラベルは英語。
- コミットメッセージ本文は日本語。
- 図はMermaidで書く。ASCIIアートの図は使わない。

## 開発スタイル

- コミットは細かく(1コミット=1論理変更)、Conventional Commitsに従う。
- `main`への直接コミットはしない。ブランチ+PRで進める。
- TDD(t_wada流)で進める: Red、Green、Refactor。
- コミット・プッシュしただけで「完了」と報告しない。CI通過・動作確認まで確認してから完了と伝える。確認できないときは「プッシュ済み、CIの確認待ち」と言う。

## ツールの好み

- Pythonはpipではなくuvエコシステムを使う(`uv sync --locked`・`uv run pytest`。CI/CDでは必ず`--locked`)。
- 新しい依存パッケージの追加は慎重に行う。サプライチェーン対策(Dependabotの`cooldown`、uvの`exclude-newer = "7d"`など)を私の承認なしに外さない。

## 環境

- 主な開発環境はWindows 11(PowerShell)とWSL2。パスやコマンド例はどちらの環境かを確認してから提示する。

## 新しいリポジトリを始めるとき

- エージェント設定のテンプレート: https://github.com/suchi/suagtemplate
- リポジトリに`AGENTS.md`がない場合は、テンプレートからの導入を提案する。
