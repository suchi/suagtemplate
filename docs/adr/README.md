# ADR(アーキテクチャ決定記録)

技術選択・設計判断を記録する。1決定=1ファイルで、`NNNN-title.md`(番号は4桁ゼロ埋めの連番、タイトルは英語kebab-case)とする。本文は日本語で書く。

- テンプレート: [0000-template.md](0000-template.md)
- 運用ルールの詳細: リポジトリルートの`AGENTS.md`「ADR」節

## 運用ルール(要約)

- 技術選択・設計判断をしたら必ずADRを残す。
- 既存ADRの上書き・修正は禁止。決定が変わったら:
  1. 旧ADRのステータスを`採用`から`非推奨`に変え、冒頭に新ADRへのリンクと非推奨理由を追記する
  2. 次の番号で新ADRを作成し、置き換え対象の旧ADRへのリンクを記載する
  3. このREADMEの一覧を更新し、非推奨を明示する
- レビュー指摘を受けても現状維持する場合は、コードに`Why:`コメントを書き、ADRにも記録する。
