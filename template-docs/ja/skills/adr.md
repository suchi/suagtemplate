# adrスキル(日本語訳)

ADR(アーキテクチャ決定記録)を`docs/adr/`に作成・改訂するスキル。技術選択・設計判断をしたとき、レビュー提案をあえて採用しないとき、過去の決定を変えるときに使う。

ADRは`docs/adr/`に`NNNN-title.md`(4桁ゼロ埋め連番、タイトルは英語kebab-case)として置く。本文は日本語。`docs/adr/0000-template.md`がテンプレート、`docs/adr/README.md`がインデックス。

## 新規ADRの作成

1. 次の番号を決める: `docs/adr/`を一覧して最大値+1。
2. `0000-template.md`を`NNNN-title.md`にコピーし、背景・決定・結果を埋める。
3. `docs/adr/README.md`のインデックス表に追加する。

## 決定の変更(置き換え — 上書き禁止)

既存ADRの書き換え・上書きは絶対にしない。代わりに:

1. 旧ADRを非推奨化する: ステータスを`採用`から`非推奨`に変え、冒頭に新ADRへのリンクと非推奨理由を追記する。
2. 次の番号で新ADRを作成し、置き換え対象の旧ADRへのリンクを記載する。
3. `docs/adr/README.md`のインデックスを更新し、旧ADRの非推奨を明示する。

## Why:コメント

レビュー指摘を受けても技術的・環境的理由で現状を維持する場合は、該当箇所に1〜3行の`Why:`コメントを書き、ADRを参照する(例: `// Why: ... see ADR-0012`)。
