# sync-docs-enスキル(日本語訳)

日本語Markdownドキュメントの英語版(`_en.md`)を同期させるスキル。`_en.md`対応ファイルを持つ日本語mdを編集した後、check-docs-en-sync hookにリマインドされたとき、英語版が必要な新規日本語ドキュメントを作成したときに使う。

ルール(AGENTS.md): 日本語の`<名前>.md`が正であり、`<名前>_en.md`は英語版。同じコミットの中で更新しなければならない。

## 手順

1. 日本語ファイルの変更点を特定する: `git diff -- <名前>.md`(新規ならファイル全体)。
2. 同じ変更を`<名前>_en.md`に適用する:
   - 文章は自然に翻訳する。コードブロック・コマンド・ファイルパス・固有名詞は翻訳しない。
   - 2つのファイルを横に並べて比較できるよう、見出し構造とセクションの順序を一致させる。
   - 例は意図的に言語固有でない限り同一に保つ(例: コミットメッセージ例の本文は日本語のまま)。
3. `<名前>_en.md`が存在せず、対象が内容のあるドキュメントなら作成する。以下はデフォルトでは英語版を作らない:
   - 薄いポインタファイル(`CLAUDE.md`、`.github/copilot-instructions.md`)
   - 実プロジェクトの個別ADR(テンプレートに同梱される雛形ファイルは除く)
4. 両ファイルを同じコミットに含める。
