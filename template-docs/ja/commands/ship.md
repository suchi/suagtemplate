# /ship(日本語訳)

現在のブランチをPRに出せる状態にする: ドキュメント整合性チェック、残りの変更のコミット、プッシュ。

## 1. ドキュメントと実装の整合性

プロジェクトのドキュメント(README.md、AGENTS.md、docs/)を読み、実態と照合する:

- コマンド・スクリプト名(ドキュメント通りに存在し動くか)
- ドキュメントに書かれたファイルパス・ディレクトリ構成
- 依存関係の名前・バージョンとマニフェストの一致
- 機能説明と実際の挙動の一致
- 削除・リネームされたファイルへの言及が残っていないか
- 日英ペア(`<名前>.md` / `<名前>_en.md`)が同期しているか

## 2. 不整合の修正

- ドキュメント側の誤りはドキュメントを、コード側の誤りはコードを修正する。
- 何をどう修正したかをユーザーに報告する。
- 何もなければ「整合性OK」と報告して続行する。

## 3. 最終コミット

変更がある場合のみ。/commitの手順に従う(検証コマンド、Conventional Commitsメッセージ、ユーザー確認)。

## 4. プッシュ

- ユーザーの確認を得てから現在のブランチをプッシュする(上流未設定なら`git push -u origin <ブランチ>`)。
- プッシュしたブランチ名と、PRがある場合はそのCI状況を報告する。
