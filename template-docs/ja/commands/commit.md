# /commit(日本語訳)

リポジトリの規約(AGENTS.md)に従って現在の変更をコミットする。

## 手順

1. `git status`と`git diff`(ステージ済みがあれば`git diff --staged`も)で変更内容を把握する。
2. 整合性チェック(自明な変更のときだけ省略可):
   - README / AGENTS.mdに書かれたコマンド・構成が実態と一致しているか
   - コード内コメント・ドキュメントが実装と矛盾していないか
   - 技術選択・設計判断を伴う変更なら`docs/adr/`にADRがあるか
   - 日本語ドキュメント`<名前>.md`を変更した場合、`<名前>_en.md`(存在する場合)も更新済みか
   - 不整合があれば修正してからコミットに進む
3. AGENTS.mdの「検証コマンド」表のコマンド(テスト・リント・型チェック)を実行する。失敗があれば修正してから進む。表が「(未設定)」のままならユーザーに伝え、同意を得た場合のみ続行する。
4. 関連ファイルを明示的にステージし(無関係な変更があるときは`git add -A`を避ける)、`git diff --staged`を確認する。
5. Conventional Commits形式のメッセージを作成する:
   - 1行目: 英語・命令形・50文字以内(例: `feat: add filter feature`)
   - 本文: 日本語で背景・内容を説明
6. メッセージをユーザーに提示し、承認を得てからコミットする。
